AddRef()

  • In COM every time an interface is created or passed, AddRef() must be called to let the COM layer that one more instance of that interface is running. When you have stopped using that interface, don’t forget to Release it to avoid memory leaks. I.e. interface_name->AddRef(), or interface_name->Release(). When CoCreateInstance() is called, AddRef() is incremented automatically.

BSTR

·        In general, strings should be passed to APIs using ByVal. Visual Basic uses a String data type known as a BSTR, which is a data type defined by Automation (formerly called OLE Automation). A BSTR is comprised of a header, which includes information about the length of the string, and the string itself, which may include embedded nulls. A BSTR is passed as a pointer, so the DLL procedure is able to modify the string. (A pointer is a variable that contains the memory location of another variable, rather than the actual data.) BSTRs are Unicode, which means that each character takes two bytes. BSTRs typically end with a two-byte null character.

The procedures in most DLLs (and in all procedures in the Windows API) recognize LPSTR types, which are pointers to standard null-terminated C strings (also called ASCIIZ strings). LPSTRs have no prefix. The following figure shows an LPSTR that points to an ASCIIZ string.

If a DLL procedure expects an LPSTR (a pointer to a null-terminated string) as an argument, pass the BSTR by value. Because a pointer to a BSTR is a pointer to the first data byte of a null-terminated string, it looks like an LPSTR to the DLL procedure. For more see the Help file (also under ANSI and Unicode).

·        To perform copy or other operations use: SysAllocString(“”) and when you are done, SysFreeString(“”).

·        To cast from 1 byte to 2 bytes use an L prefix: ItemName = SysAllocString(L"VectorValueX");

Component does not disappear

  • If you kill DAD and you cannot delete the component from DAD, or unregister from the Console Manager, the way to fix this is by loading either a new or saved component with the same name. When you do this, it loaded component will replace the dead one in DAD, and after that you will be able to remove it or do other manipulations to it.

Component Creation Error when inserting another agent

  • Sometimes when there is an agent in DAD, and a second Agent is inserted, one will see an error message saying “Component Creation Error”, even other times “Load Data Error”, or “Binding Update Error.” If you see this happen, delete the inserted agent, and try again. Many times after the second try there will be no errors. As of July 2005, we have not understood this behavior.

 

Component Creation Error, have you checked your .lib and dll’s?

  • Sometimes your project may even compile, but when you run DAD and load the component it will say that it cannon created it. If you are using third party technology, make sure you include the library in VC++ under Tools->Project Settings->Link->Input and also make sure that all .dll’s, .libs, and .h files are in the same directory as your project in the O:\ drive.
  • There is a program called Depends.exe found at O:\System\Tools\DependencyWalker that will help you identify which .dll is giving you the problem.

 

Connecting DAD across computers

  • Perhaps you might come across the problem of connecting DAD across computers, maybe with JSC-DRL. Note that Jsc-Drl has two ethernet ports. The cable needs to be plugged into the one beneath the monitor cable. If the cable is plugged into the upper one, it does not get the correct static ip address (and therefore, none of the other DADs can connect to it). In ipconfig, you can tell there are two ports because you get two local area connection listings. The first one is the one with the correct static ip. If it's listed as "cable disconnected" you need to unplug and re-plug the cable.

Debugging

  • BSTRINGS –  To show BSTRINGS in the debugger Watch Window, go to: Tools->Options->Debug and check Display Unicode Strings.
  • Debugging Components – Make sure that when you insert a component to DAD for debugging purposes you UNCHECK “Create in Parent’s Process.” This will allow you to step through your code. By doing this, it will run in DAD’s process. Whenever the debugger runs, it only does so in the process that contains it. If we clicked on that check box, a separate process would be created and it would not be possible to trace it.
    • When debugging in DAD and you are done with it, MAKE SURE YOU MANUALLY REMOVE ALL COMPONENTS FROM DAD. Otherwise, DAD terminates but the pointers to the components do not.
  • OLE VIEW – Sometimes using OLE View as the executable for the debug session can be useful. If you are able to step through your code, you know that the .dll version that you are using is correct.
  • VB – If you want to debug the actual DAD code, or HelperX code, you can set VB as the executable for the debug session. Run the debugger, Open DAD, Hit Ctrl+Break, and then look at the location of the error. This procedure allows you to enter to code you would not be able to if you just had DAD as the executable for the debugger.
  • CRASH – Sometimes when you are debugging the code and the program crashes for some reason, you might notice that the next time you try to open DAD it will say that you do not have this ‘functionality’. When the program crashes while debugging it messes with the registration of the program. Type going to C:\IMA2\System\Framework and double click on 1) HelperX.ocx, 2) DadX.ocx, and 3) DAD.exe. DAD should be able to run. Otherwise you will have to open the VisualBasic VPG O:\Source\Framework\Dad_DadX_HelperX.vpg. First make HelperX, then make DadX, then make DAD. Make sure that when you make these, the created files have the C:\IMA2\System\Framework as their source folder. DAD should then work.

Installing IMA 2.5

  • IMA2_Install.bat – Sometimes the batch file’s line may not be pointing to the right address in your computer. Right click on the file, and select the Edit option. Make sure that the address on the first line points to the right address.
  • Call Command – Sometimes the standard method “Call” found in the IMA2_Install.bat may not be registered in your computer. Go to the command prompt, and go to C:\Windows and run regsvr32 for this method.
  • Using IMA2_Install.bat – this file calls the IMA2_Register.bat file, which registers all the IMA 2.5 binaries in your computer. A log file is created in the root directory of IMA 2.5.

Manager Book – Opening?

  • Sometimes when you try to open the Manager Book for the first time you will find problems with loading the Active X controls. This is a fault in VB. Sometimes you can open the Manager Book.vbp with Notepad. Delete the HelperX.dll statement. Save the file and close it. Reregister the HelperX.ocx and Helper.dll. Open the Manager Book.vbp and add IMA2_HelperX.ocx under Project->Component; and IMA2_Helper.dll under Project->References.
  • Sometimes ManagerBook will not open properly if you have not installed the latest version of MDAC. There should be a recent version of MDAC under IMA2\Downloads.

Properties in IMA 2.5 components vs. IMA 1.0 components

  • You will notice that for some components in IMA 2.5 there is an additional property compared to some IMA 1.0 components. In the latter, some properties contained two arguments, namely, an index and an array. It was created with manual debugging in mind. In this way, through AgentBuilder, one could change the value of the array for a given index. In IMA 2.5, the user interface is different and multiple argument lists for a property cannot be read. It is for this reason, that the index parameter of the argument list, is added as a new property in IMA 2.5.

VB Problems

  • Binary Compatibility / CLSID / IID changes / DLL is different – In VB if the project is not set to Binary Compatibility, any time a change is made to the project and it is updated, it will created a new IID for the dll. If any other computer is using a previous version, there will be conflicts. By having binary compatibility, this problem is resolved. To do that, go to Project->Properties->Make->choose Binary Compatibility.
