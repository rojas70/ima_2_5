Automatic Documentation Macro for Developer Studio

I wrote a VBScript macro for DevStudio. This macro operates on a header file and produces an html file. The name of the output html file is headerfilenameADF.html. So if you had a header file named MyHeader.h, the output will be MyHeaderADF.html.

How to Use the Macro

The macro file is: U:\IRLUSERS\alford\exec\CompDoc.DSM

Copy this file to the directory: D:\Program Files\DevStudio\SharedIDE\Macros

Go to the header file for the class you want to document. Run the macro. (To install and run the macro, see the DevStudio Infoviewer pages below.) That's it.

What the Macro Does

The macro runs through the header file 3 times. It makes a "skeleton" document, which contains:
  • Class Name
  • Source Code Location
  • COM Interace Map Entries
  • Inteface Methods and Properties (note: properties are not listed as such; instead, the get and put methods for the properties are listed).
  • Member Functions
Interface Methods and Member Functions both have a section for description and a list of parameters with their types. Member Functions also have their return types listed.

Caveats

The macro isn't perfect. Any function calls in the header file (such as SetupComponentLinks) will get stuck in the output file as member functions. Also, beware of really long header files (greater than 1000 lines). There are probably other bugs as well. Let me know when you find them.
